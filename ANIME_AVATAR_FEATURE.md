**数据库字段：**
```sql
-- secretary_avatars 表
avatar_url TEXT       -- 存储头像图片的完整 URL（圆形头像用）
full_body_url TEXT    -- 存储3D立绘图片的完整 URL（3D全身展示用）
```

### 图片显示

**React 组件实现：**

```typescript
// 获取头像图片（用于最小化状态）
const getAvatarImage = () => {
  if (!config?.avatar || !config.avatar.avatar_url) return null;
  return config.avatar.avatar_url;
};

// 获取全身立绘图片（用于展开状态）
const getFullBodyImage = () => {
  if (!config?.avatar || !config.avatar.full_body_url) return null;
  return config.avatar.full_body_url;
};

// 显示全身立绘
{getFullBodyImage() ? (
  <div className="w-32 h-48 flex items-end justify-center overflow-hidden rounded-lg bg-gradient-to-b from-primary/5 to-secondary/10">
    <img 
      src={getFullBodyImage()!} 
      alt={config.name}
      className="h-full w-auto object-contain object-bottom"
    />
  </div>
) : (
  // 回退到头像显示
  <div className="w-16 h-16 rounded-full overflow-hidden">
    <img src={getAvatarImage()!} alt={config.name} />
  </div>
)}
```

**样式处理：**
```css
/* 全身立绘容器 */
.full-body-container {
  width: 128px;
  height: 192px;
  border-radius: 8px;
  overflow: hidden;
  background: linear-gradient(to bottom, 
    hsl(var(--primary) / 0.05), 
    hsl(var(--secondary) / 0.1));
}

/* 图片填充 - 保持比例，底部对齐 */
.full-body-image {
  height: 100%;
  width: auto;
  object-fit: contain;
  object-position: bottom;
}
```

## 常见问题

### Q1: 如何更换秘书形象？

A: 进入设置页面 → 专属学习秘书 → 形象标签 → 点击想要的形象卡片 → 保存配置

### Q2: 形象图片加载慢怎么办？

A: 图片存储在 CDN 上，通常加载很快。如果网络较慢，请稍等片刻。图片会自动缓存，下次加载更快。

### Q3: 可以自定义上传形象图片吗？

A: 目前暂不支持自定义上传。我们提供了6种精心挑选的动漫形象供选择。未来版本会考虑添加自定义功能。

### Q4: 形象图片会占用很多流量吗？

A: 不会。每张图片都经过优化，大小适中。而且图片会被浏览器缓存，只需加载一次。

### Q5: 形象图片在哪些地方显示？

A: 形象图片会在以下位置显示：
1. 设置页面的形象选择卡片
2. 仪表盘的秘书问候卡片
3. 浮窗秘书（展开和最小化状态）

### Q6: 不同形象的图片风格一致吗？

A: 是的。所有形象都采用统一的动漫风格，确保视觉一致性和美观度。

### Q7: 形象图片会定期更新吗？

A: 目前的形象是精心挑选的经典动漫风格。未来可能会根据用户反馈和节日主题更新形象。

### Q8: 形象图片版权有问题吗？

A: 所有形象图片都通过正规渠道获取，确保版权合规。用户可以放心使用。
